# Security Policy

## Supported Versions

Chronos is currently in active development (pre-1.0 phase). Only the most recent minor version receives security updates. We strongly encourage all users to run the latest version available on the `main` branch or the most recent release.

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| < 0.1   | :x:                |

## Reporting a Vulnerability

Because Chronos is a personal context engine that processes sensitive screen data and interacts with local Vision Language Models (VLMs), we treat any security flaw, sandbox escape, or privacy leak with the highest priority.

**Please do not report security vulnerabilities through public GitHub issues.**

To report a vulnerability, please use one of the following options:
1. **GitHub Security Advisories:** Navigate to the "Security" tab in this repository and click "Report a vulnerability" to open a private communication channel with the maintainers.
2. **Direct Email:** Send a detailed email to `[YOUR_EMAIL_HERE]`.

**What to expect:**
* You will receive an acknowledgment of your report within 48 hours.
* We will triage the issue to verify its validity and its impact on the system's air-gapped architecture.
* We will keep you informed about the progress of the patch.
* Once the vulnerability is resolved and a patch is released, we will provide proper credit to you (if desired) in the Release Notes.

Thank you for helping keep Chronos secure and privacy-focused.
